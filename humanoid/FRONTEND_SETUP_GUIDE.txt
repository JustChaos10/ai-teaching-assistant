2. LIVE2D MODEL DOWNLOAD & SETUP
================================================================================

The Live2D Shizuka model is NOT included in the repository due to licensing.
You must download it separately.

OPTION A: Download from Live2D Official (RECOMMENDED)
------------------------------------------------------

Step 1: Visit Live2D Sample Data Page
   URL: https://www.live2d.com/en/download/sample-data/

Step 2: Find "Shizuku" Sample Model
   - Look for "Shizuku (しずく)" in the list
   - Click the download button
   - File: Usually named "Shizuku.zip" or similar

Step 3: Extract the ZIP file
   - Extract to a temporary location
   - You should see folders like:
     • shizuku.model.json
     • textures/
     • motions/
     • physics/

Step 4: Copy Model Files to Project
   Copy ALL extracted files to:

   /home/user/ai-teaching-assistant/humanoid/r3f-lipsync-tutorial/public/live2d-models/shizuku/

   Expected structure:
   public/
   └── live2d-models/
       └── shizuku/
           ├── shizuku.model.json       (REQUIRED)
           ├── shizuku.moc              (REQUIRED)
           ├── textures/
           │   ├── texture_00.png       (REQUIRED)
           │   └── (other .png files)
           ├── motions/                 (Optional)
           └── expressions/             (Optional)


OPTION B: Use Free Resources
-----------------------------

Alternative sources for Live2D models:
1. GitHub: Search for "live2d sample models"
2. Open-LLM-VTuber: https://github.com/Open-LLM-VTuber/Open-LLM-VTuber
   (Check their live2d-models folder)

⚠️  IMPORTANT: Ensure compliance with Live2D Free Material License Agreement
   https://www.live2d.com/en/terms/live2d-free-material-license-agreement/


VERIFICATION:
-------------

After setup, verify these files exist:

cd /home/user/ai-teaching-assistant/humanoid/r3f-lipsync-tutorial
ls -la public/live2d-models/shizuku/

You should see:
-rw-r--r-- shizuku.model.json
-rw-r--r-- shizuku.moc
drwxr-xr-x textures/

If files are missing, the avatar will NOT load!

7. TROUBLESHOOTING COMMON ISSUES
================================================================================

Issue 1: "npm install" fails
-----------------------------

Error: EACCES permission denied

Solution:
sudo chown -R $USER:$USER ~/.npm
sudo chown -R $USER:$USER /home/user/ai-teaching-assistant/humanoid
npm install


Issue 2: Live2D Model Not Loading
----------------------------------

Error in console: "Failed to load Live2D model"

Solutions:
1. Verify model path:
   ls public/live2d-models/shizuku/shizuku.model.json

2. Check file permissions:
   chmod -R 755 public/live2d-models/

3. Try different model path in Live2DAvatar.jsx:
   "/live2d-models/shizuku/shizuku.model3.json" (for Cubism 3)
   "/live2d-models/shizuku/shizuku.model.json" (for Cubism 2)


Issue 3: CORS Errors
--------------------

Error: "Access-Control-Allow-Origin"

Solution:
Backend should already allow CORS. If not, verify backend/app.py:

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)


Issue 4: Avatar Too Large/Small
--------------------------------

Solution:
Edit src/components/Live2DAvatar.jsx:

// Around line 73-74
const scaleX = (app.screen.width * 0.8) / model.width;  // Change 0.8
const scaleY = (app.screen.height * 0.9) / model.height; // Change 0.9

Increase = larger avatar
Decrease = smaller avatar


Issue 5: No Audio Playback
---------------------------

Solutions:
1. Check browser audio not muted
2. Check system volume
3. Verify audio file exists:
   curl http://127.0.0.1:8000/audio/[filename].wav
4. Check browser console for audio errors


Issue 6: Microphone Not Working
--------------------------------

Solutions:
1. Grant microphone permissions in browser
2. Check browser security settings
3. Use HTTPS or localhost (required for mic access)
4. Test with: navigator.mediaDevices.getUserMedia({audio:true})


Issue 7: Vite Port Already in Use
----------------------------------

Error: "Port 5173 is already in use"

Solution:
Kill existing process:
lsof -ti:5173 | xargs kill -9

Or use different port:
npm run dev -- --port 5174
